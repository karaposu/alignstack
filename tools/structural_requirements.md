# Structural Requirements

Requirements per discipline. Format: `"grep pattern" | human-readable label`
The structural_check.sh tool reads this file and checks discipline output against these patterns.

## sensemaking
- "SV1" | SV1 baseline understanding
- "SV6" | SV6 stabilized model
- "Anchor Extraction" | Cognitive anchor extraction phase
- "Perspective" | Perspective checking phase
- "Ambiguity Collapse" | Ambiguity collapse phase
- "Degrees-of-Freedom" | Degrees-of-freedom reduction phase
- "Saturation Indicators" | Saturation indicators telemetry

## innovation
- "Mechanism" | At least one mechanism applied
- "Assembly" | Assembly check
- "Mechanism Coverage" | Mechanism coverage telemetry
- "SURVIVE|KILL|REFINE" | At least one tested output with verdict

## critique
- "Dimension" | Evaluation dimensions defined
- "Prosecution" | Prosecution (adversarial case against)
- "Defense" | Defense (adversarial case for)
- "Verdict" | Verdict rendered
- "Convergence Telemetry" | Convergence telemetry

## exploration
- "Cycle" | At least one exploration cycle
- "Signal" | Signal detection
- "Convergence" | Convergence assessment
- "Confidence" | Confidence map or confidence levels
- "Frontier" | Frontier tracking

## decomposition
- "Coupling" | Coupling topology perception
- "Boundar" | Boundary detection
- "Question" | Question tree
- "Interface" | Interface map
- "Dependen" | Dependency ordering
- "Self-Evaluate|Self-Eval" | Self-evaluation
